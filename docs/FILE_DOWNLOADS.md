# File Downloads

The Running Coach Service allows you to save and download AI-generated files such as training plans, frameworks, and calendars.

## Features

- **Save Chat Responses** - Save any AI response as a downloadable file
- **Browse Files** - View all saved files organized by category
- **Download Files** - Download files in their original format (Markdown, ICS, etc.)
- **File Management** - List, download, and delete files via API or web interface

## Using the Web Interface

### Saving a Chat Response

1. Chat with your coach to get training advice
2. Click the **💾 Save as File** button below any AI response
3. Enter a filename (extension is added automatically)
4. File is saved and available in the Files tab

### Browsing and Downloading Files

1. Click the **Files** tab in the web interface
2. Browse available files by category:
   - **Training Plans** - Periodized training schedules
   - **Frameworks** - Training methodology templates
   - **Calendars** - ICS calendar files
3. Click **⬇ Download** to download any file

### Filtering Files

Use the filter buttons to view files by category:
- **All Files** - Show everything
- **Training Plans** - Show only plans
- **Frameworks** - Show only frameworks
- **Calendars** - Show only calendar files

## Using the API

### List Files

**GET** `/api/files`

Query Parameters:
- `category` (optional): Filter by category (plans, frameworks, calendar)

Response:
```json
{
  "files": [
    {
      "filename": "marathon_plan_2025.md",
      "category": "plans",
      "size": 4096,
      "modified": "2025-01-15T10:30:00",
      "path": "plans/marathon_plan_2025.md"
    }
  ]
}
```

### Download a File

**GET** `/api/files/<category>/<filename>`

Example:
```bash
curl -O http://localhost:5000/api/files/plans/marathon_plan_2025.md
```

### Save a New File

**POST** `/api/files`

Request Body:
```json
{
  "content": "# My Training Plan\n\n...",
  "filename": "my_plan.md",
  "category": "plans",
  "metadata": {
    "agent": "running-coach",
    "created_at": "2025-01-15T10:30:00"
  }
}
```

Response:
```json
{
  "success": true,
  "filename": "my_plan.md",
  "category": "plans",
  "path": "/app/data/plans/my_plan.md"
}
```

### Delete a File

**DELETE** `/api/files/<category>/<filename>`

Example:
```bash
curl -X DELETE http://localhost:5000/api/files/plans/old_plan.md
```

## File Categories

### Training Plans (`plans`)

Periodized training schedules generated by the running coach:
- Multi-week training plans
- Race-specific preparation
- Base building programs
- Recovery protocols

### Frameworks (`frameworks`)

Training methodology templates:
- Post-marathon recovery frameworks
- Periodization templates
- Training phase structures
- Workout progression models

### Calendars (`calendar`)

ICS calendar files for external calendar apps:
- Scheduled workouts
- Training schedule exports
- Race calendars

## File Formats

### Markdown (.md)

Most training plans and frameworks are saved as Markdown:
- Human-readable text format
- Supports formatting (headings, lists, bold, italic)
- Can be opened in any text editor
- Can be imported into note-taking apps

### ICS Calendar (.ics)

Calendar files for importing into calendar apps:
- Standard iCalendar format
- Compatible with Google Calendar, Outlook, Apple Calendar
- Contains workout events with dates and descriptions

## File Storage

Files are stored in the Docker container at:
- Training Plans: `data/plans/`
- Frameworks: `data/frameworks/`
- Calendars: `data/calendar/`

These directories are mounted as volumes, so files persist across container restarts.

## Python Example

```python
import requests

BASE_URL = "http://localhost:5000"

# Save a file
response = requests.post(f"{BASE_URL}/api/files", json={
    "content": "# Marathon Training Plan\n\nWeek 1...",
    "filename": "marathon_plan.md",
    "category": "plans",
    "metadata": {
        "agent": "running-coach",
        "race": "Marathon 2025"
    }
})

print(response.json())

# List all files
response = requests.get(f"{BASE_URL}/api/files")
files = response.json()['files']

for file in files:
    print(f"{file['filename']} - {file['category']}")

# Download a specific file
response = requests.get(
    f"{BASE_URL}/api/files/plans/marathon_plan.md"
)

with open("downloaded_plan.md", "w") as f:
    f.write(response.text)
```

## JavaScript Example

```javascript
// Save a file
async function saveFile(content, filename, category) {
    const response = await fetch('/api/files', {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json',
        },
        body: JSON.stringify({
            content: content,
            filename: filename,
            category: category,
            metadata: {
                created_at: new Date().toISOString()
            }
        })
    });

    return await response.json();
}

// List files
async function listFiles(category = null) {
    const url = category
        ? `/api/files?category=${category}`
        : '/api/files';

    const response = await fetch(url);
    const data = await response.json();
    return data.files;
}

// Download a file
async function downloadFile(category, filename) {
    const response = await fetch(`/api/files/${category}/${filename}`);
    const blob = await response.blob();

    const url = window.URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = filename;
    document.body.appendChild(a);
    a.click();
    window.URL.revokeObjectURL(url);
    document.body.removeChild(a);
}
```

## Tips

1. **Descriptive Filenames** - Use clear, descriptive names with dates
   - Good: `marathon_plan_spring_2025.md`
   - Bad: `plan1.md`

2. **Organize by Category** - Save files in the appropriate category
   - Training plans → `plans`
   - Training frameworks → `frameworks`
   - Calendar exports → `calendar`

3. **Regular Exports** - Download important files to your local machine for backup

4. **Version Control** - Include dates in filenames to track different versions
   - `base_building_v1_2025-01-15.md`
   - `base_building_v2_2025-02-01.md`

## Security Notes

- Files are only accessible within the Docker container's network
- Use a reverse proxy with authentication for production deployments
- File operations check for path traversal attempts
- Maximum filename length is 255 characters

## Troubleshooting

### File Not Found

- Check the category and filename are correct
- Use the `/api/files` endpoint to list available files
- Verify the file wasn't deleted

### Save Failed

- Check that the filename doesn't contain invalid characters
- Ensure the category is valid (plans, frameworks, calendar)
- Verify the container has write permissions to the data directory

### Download Issues

- Check browser download settings
- Ensure popup blockers aren't preventing downloads
- Try downloading via API directly with `curl`
